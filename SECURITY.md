# Security Policy

## Reporting a Vulnerability

The security of the Sovereign AI Fleet project is important to us. If you discover a security vulnerability in this documentation hub or any of the related MCP servers, we appreciate your help in disclosing it responsibly.

### Private Disclosure Process

**Please do NOT report security vulnerabilities through public GitHub issues.**

Instead, please report security vulnerabilities privately to:

- **Contact**: ki7mt
- **Method**: Use GitHub's private vulnerability reporting feature or contact directly through secure channels

### What to Include

When reporting a vulnerability, please include:

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Any suggested fixes or mitigations (if available)
- Your contact information for follow-up

### Response Timeline

- We will acknowledge receipt of your vulnerability report within 48 hours
- We will provide a detailed response indicating next steps within 7 days
- We will keep you informed of our progress toward a fix
- We will notify you when the vulnerability has been addressed

### Scope

This security policy applies to:

- The ki7mt-ai-docs documentation hub
- Configuration files and build scripts in this repository
- Related MCP server repositories (wspr-mcp, adif-mcp, solar-mcp)

### Safe Harbor

We support safe harbor for security researchers who:

- Make a good faith effort to avoid privacy violations and data destruction
- Only interact with accounts you own or with explicit permission
- Do not exploit a vulnerability beyond the minimal testing required
- Report vulnerabilities promptly
- Keep vulnerability details private until they are resolved

## Security Best Practices

When contributing to this project:

- Do not commit sensitive information (API keys, passwords, tokens)
- Review dependencies for known vulnerabilities
- Follow secure coding practices
- Keep dependencies up to date

## Additional Information

For questions about this security policy, please open an issue in this repository (for non-sensitive questions only) or contact ki7mt directly for sensitive matters.

Thank you for helping keep the Sovereign AI Fleet project secure!
